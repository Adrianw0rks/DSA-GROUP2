Sorting Algorithms Visualizer

Bubble Sort

  • Time Complexity:
    • Best Case: O(n) - if the list is already sorted.
    • Average/Worst Case: O(n2) - nested iterations.
  • Space Complexity: O(1) - it's an in-place algorithm, meaning it uses constant space.
  • Implication: Best used for small datasets or educational purposes due to its simplicity.

Selection Sort

  • Time Complexity:
    • Best, Average, Worst Case: O(n2) - nested loops.
  • Space Complexity: O(1) - in-place algorithm.
  • Implication: Ideal for small lists or limited memory usage available.

Insertion Sort

  • Time Complexity:
    • Best Case: O(n) - list is already sorted.
    • Average/Worst Case: O(n2) - shifting elements for insertion.
  • Space Complexity: O(1) - in-place algorithm.
  • Implication: Best for nearly sorted or small datasets.

Merge Sort

  • Time Complexity:
    • Best, Average, and Worst Case: O(n log n) - recursive splitting and merging.
  • Space Complexity: O(n) - extra space for merging.
  • Implication: Best for datasets that require stable sorting as it ensures consistent performance.

Quick Sort

  • Time Complexity:
    • Best and Average Case: O(n log n) - good pivot selection.
    • Worst Case: O(n2) - pivot divides the lists poorly.
  • Space Complexity: O(log n) - recursive stack.
  • Implication: Ideal for large datasets where average-case performance of O(n log n) is needed and space efficiency is important.
